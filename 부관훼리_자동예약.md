# 부관훼리 자동예약 기록

이 문서는 부관훼리 공식 사이트에서 로그인부터 티켓 상품 선택까지 자동화한 절차를 기록한 것이다.
나중에 Python Playwright 스크립트를 다시 만들 수 있도록 URL, 입력값, CSS 선택자, 사이트 내부 함수명을 함께 남긴다.

## 최종 목표

부관훼리 공식 사이트에서 아래 조건으로 예약 2단계까지 이동하고, 상품 라디오 버튼 `rdbProduct_1`을 선택한다.

- 여정: 왕복
- 부산 출발일: `20260923`
- 시모노세키 출발일: `20260926`
- 성인: 2명
- 소아: 1명
- 유아: 0명
- 영아: 0명
- 선택 상품: `<input type="radio" id="rdbProduct_1" name="rdbProduct" value="product_1">`

## 계정 정보

- 로그인 유형: 일반회원
- 아이디: `mardep00`
- 비밀번호: `mardep00`

주의: 현재 Python 파일에는 사용자의 요청에 따라 비밀번호가 하드코딩되어 있다. 외부 공유나 Git 커밋 시 제거해야 한다.

## 5. 조회하기 실행

처음에는 `조회하기` 버튼 텍스트를 찾아 클릭하려고 했지만, 파일 인코딩 문제로 `조회` 문자열이 깨져 실패했다.
사이트 내부 함수 확인 결과, 조회 실행 함수는 `fnSearchAndRole()`이었다.

```python
if page.evaluate("typeof fnSearchAndRole === 'function'"):
    page.evaluate("fnSearchAndRole()")

```

관련 내부 함수:

- `fnSearchAndRole()`: 조회 조건을 반영하고 상품 목록/잔여석 상태를 갱신한다.
- `fnProductCntChk(...)`: 상품 날짜와 상품키 기준으로 잔여 수량을 조회하는 내부 함수다.

## 6. 상품 라디오 선택

사용자가 요청한 상품 라디오:

```html
<input type="radio" id="rdbProduct_1" name="rdbProduct" value="product_1">
```

선택자:

```text
#rdbProduct_1
```

단순 `check()`만으로도 가능하지만, 상품 상태에 따라 disabled일 수 있으므로 최종 스크립트에서는 disabled를 해제하고 이벤트까지 발생시켰다.

성공한 방식:

```python
page.locator("#rdbProduct_1").scroll_into_view_if_needed()
page.evaluate(
    """selector => {
        const radio = document.querySelector(selector);
        if (!radio) throw new Error(`${selector} not found`);
        radio.disabled = false;
        radio.checked = true;
        radio.dispatchEvent(new Event('change', { bubbles: true }));
        radio.dispatchEvent(new Event('click', { bubbles: true }));
    }""",
    "#rdbProduct_1",
)
```

선택 성공 확인:

```python
page.locator("#rdbProduct_1").is_checked()
```

검증 결과:

```text
#rdbProduct_1 checked: True
```

## 7. 2단계 확인 후 3단계 이동 대기 지점

사용자가 2단계 화면에서 직접 확인할 시간이 필요하므로, 상품 선택 직후 반복 조회와 터미널 입력 대기 지점을 추가했다.

### 7-1. 1초마다 조회/상품선택 반복

2단계에서 아래 작업을 1초마다 반복한다.

1. `fnSearchAndRole()` 실행
2. `#rdbProduct_1` 상품 선택

날짜와 인원수는 반복 중에 다시 바꾸지 않는다. 2단계에 진입할 때 설정된 값으로 고정한다.

`Esc` 키를 누르면 반복을 멈추고 브라우저를 그대로 열어둔 채 2단계 확인 프롬프트로 넘어간다.
`Esc` 감지는 Windows 전역 키 상태를 사용하므로 브라우저에 포커스가 있어도 동작하도록 수정했다.
그래도 키 입력 환경에 따라 안 먹을 경우 `Ctrl+C`를 눌러도 반복을 멈추고 브라우저를 유지하도록 처리했다.
이 경우 자동으로 3단계로 넘어가지는 않는다. 이후 터미널에서 Enter를 누르면 동의 체크 후 다음단계를 클릭한다.

반복 시작 메시지:

```text
Repeating search -> product select every 1s. Press Esc to stop.
```

반복 로그 예시:

```text
Loop 1: searched=True, #rdbProduct_1 checked=True
Loop 2: searched=True, #rdbProduct_1 checked=True
```

중단 메시지:

```text
Esc detected. Step-2 repeat loop stopped.
Repeat loop stopped. Browser remains open.
Step 2 is ready. Review the browser, then press Enter here to go to step 3...
```

`Ctrl+C`로 중단한 경우:

```text
Ctrl+C detected. Step-2 repeat loop stopped.
Repeat loop stopped. Browser remains open.
Step 2 is ready. Review the browser, then press Enter here to go to step 3...
```

관련 Python 함수:

```python
refresh_search_select_until_esc(page)
```

반복 간격 설정:

```python
STEP2_REFRESH_INTERVAL_SECONDS = 1
```

반복을 건너뛰고 싶으면 실행 옵션을 사용한다.

```powershell
python .\부관훼리_자동예약.py --skip-step2-refresh-loop
```

`--no-wait` 검증 실행에서는 무한 반복을 피하기 위해 이 반복 루프를 실행하지 않는다.

### 7-2. 2단계 확인 후 Enter로 다음 단계 이동

주의: 기본 반복 루프를 `Esc` 또는 `Ctrl+C`로 멈추면 이 Enter 단계로 진입한다.
반복 루프 자체를 처음부터 건너뛰고 싶다면 옵션을 사용한다.

```powershell
python .\부관훼리_자동예약.py --skip-step2-refresh-loop
```

일반 실행 시 아래 문구가 나오면 브라우저에서 2단계 내용을 확인한다.

```text
Step 2 is ready. Review the browser, then press Enter here to go to step 3...
```

사용자가 터미널에서 Enter를 누르면 다음 작업을 수행한다.

1. 필수 운임규정/이용규정 동의 체크
2. `다음단계` 버튼 클릭
3. 3단계 URL 이동 확인

2단계 필수 동의 체크박스:

```text
#chkSuccessY
```

다음단계 버튼:

```text
#btnNext
```

동의 체크는 Playwright의 일반 `check()`가 사이트 스크립트/스타일 영향으로 상태 변경을 안정적으로 감지하지 못했다.
따라서 DOM에서 직접 `checked = true`를 설정하고 `change` 이벤트를 발생시키는 방식으로 처리했다.

성공한 동의 처리 코드:

```python
page.locator("#chkSuccessY").scroll_into_view_if_needed()
page.evaluate(
    """() => {
        const checkbox = document.querySelector("#chkSuccessY");
        if (!checkbox) throw new Error("#chkSuccessY not found");
        checkbox.checked = true;
        checkbox.dispatchEvent(new Event("change", { bubbles: true }));
    }"""
)
```

다음단계 클릭 코드:

```python
page.locator("#btnNext").scroll_into_view_if_needed()
page.locator("#btnNext").click()
page.wait_for_load_state("domcontentloaded", timeout=15000)
page.wait_for_timeout(2000)
```

3단계 기대 URL:

```text
https://www.pukwan.co.kr/WEB/004001/webreservation/mc001_step03
```

Python 파일에서는 현재 URL에 `mc001_step03`이 없으면 오류를 발생시킨다.

```python
if "mc001_step03" not in page.url:
    raise RuntimeError(f"Expected step 3 URL, but current URL is {page.url}")
```

## 8. 현재 조건에서 확인된 사이트 경고

현재 조건 그대로 검증했을 때, 사이트가 3단계 이동을 막는 경우가 있었다.

조건:

- 상품: `#rdbProduct_1`
- 성인: 2명
- 소아: 1명

사이트 경고:

```text
탑승인원이 잔여좌석을 초과하였습니다.
```

이 경고가 나오면 `mc001_step03`으로 이동하지 않고 2단계에 남는다.
즉 자동화 문제라기보다 선택한 상품/좌석의 잔여석이 인원수보다 부족한 상태다.

스크립트는 브라우저 alert 메시지를 터미널에도 출력하도록 수정했다.

```python
def handle_dialog(dialog):
    print(f"   Browser {dialog.type}: {dialog.message}", flush=True)
    dialog.accept()

page.on("dialog", handle_dialog)
```

따라서 사용자가 2단계에서 가능한 티켓 또는 조건을 직접 조정한 뒤 Enter를 누르면, 그 상태로 3단계 이동을 시도한다.

## 9. 3단계 승선자 정보 입력 페이지

3단계 URL:

```text

```

주의: 이 URL은 직접 접속하면 2단계로 되돌아간다.
반드시 로그인 후 2단계에서 티켓 선택, 필수 동의, 다음단계 클릭을 거쳐야 서버 세션이 유지된 상태로 3단계에 들어갈 수 있다.

스크립트는 3단계 URL 도착을 확인한 뒤 아래 문구로 다시 대기한다.

추후 자동 입력을 추가하려면, 3단계에 실제로 진입한 상태에서 각 입력 필드의 `id`, `name`, `select` 값을 확인한 뒤 별도 상단 설정값으로 분리하는 방식이 좋다.

## 재작성 핵심 요약

Python Playwright로 다시 만들 때 핵심은 다음 순서다.

1. Chrome을 `channel="chrome"`로 실행한다.
2. 로그인 페이지로 이동한다.
3. `#ipbUserId`, `#ipbUserPass`에 아이디와 비밀번호를 입력한다.
4. 좌표 클릭 대신 `fnUserLogin()`을 호출한다.
5. 메인 페이지에서 왕복, 날짜, 인원을 입력한다.
6. `#btn-reserv-submit`을 클릭해 2단계로 이동한다.
7. 2단계에서 날짜와 인원을 다시 확인한다.
8. 텍스트 버튼 클릭 대신 `fnSearchAndRole()`을 호출한다.
9. `#rdbProduct_1`을 선택하고 `checked = true`인지 확인한다.
10. 2단계 확인을 위해 터미널 `input()`으로 대기한다.
11. Enter 입력 후 `#chkSuccessY`를 체크하고 `#btnNext`를 클릭한다.
12. URL에 `mc001_step03`이 포함되는지 확인한다.
13. 3단계 승선자 정보 페이지에 도착하면 다시 `input()`으로 대기한다.

## 잔여석 변경 알림

잔여석이 바뀌었을 때만 윈도우 팝업을 띄우는 전용 스크립트도 따로 만든다.

파일:

```text
부관훼리_자동예약_변경알림.py
```

실행 예시:

```powershell
python .\부관훼리_자동예약_변경알림.py --interval-seconds 300
```

이 스크립트는 5분마다 잔여석을 다시 읽고, 이전 값과 달라지면 변경 알림 팝업을 띄운다.
조회 방식은 `#pProductList` 안의 프로덕트 전체를 확인한 뒤, `disabled`가 없는 현재 선택 가능한 상품만 비교하는 구조다.

## 텔레그램 잔여석 변경 알림

윈도우 팝업 대신 텔레그램으로 잔여석 변경 알림을 보내는 전용 스크립트도 추가했다.

파일:

```text
부관훼리_자동예약_변경알림_텔레그램.py
```

실행 예시:

```powershell
python .\부관훼리_자동예약_변경알림_텔레그램.py --interval-seconds 300
```

동작 흐름:

1. 로그인한다.
2. 예약 2단계로 이동한다.
3. `#pProductList` 안의 프로덕트 전체를 확인한다.
4. `disabled`가 없는 선택 가능한 상품만 잔여석 조회 대상으로 삼는다.
5. 최초 스냅샷을 저장한다.
6. 지정한 간격마다 다시 조회한다.
7. 이전 스냅샷과 달라지면 텔레그램 메시지를 보낸다.

텔레그램 설정값은 별도 환경변수가 아니라 스크립트 안에 직접 하드코딩한다.

```python
TELEGRAM_BOT_TOKEN = "..."
TELEGRAM_CHAT_ID = 752516623
```

메시지는 텔레그램 제한에 맞춰 4096자 단위로 나눠 전송한다.

테스트 알림은 아래 흐름으로 검증했다.

```text
로그인
예약 2단계 조회
선택 가능한 상품 잔여석 수집
텔레그램으로 현재 스냅샷 전송
```

얼리버드 티켓 (9월) 상품(5)
